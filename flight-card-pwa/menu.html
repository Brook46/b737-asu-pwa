📅 Start here — get your roster in

Your monthly bulletin is the fastest way to fill the app. One paste and every leg of the duty period lands in your leg list, sorted by UTC departure.

  1. Copy your duty bulletin Open the e-bulletin (Slip details / Cockpit format) and select-all → copy the text.
  2. Open Flight Card Tap the ✈ New Flight icon in the header.
  3. Tap "Paste roster" Paste into the box and tap Parse.
  4. Done Every leg shows up in the switcher with dep / arr / flight # / tail / schedule. The current-time leg is auto-selected; tap 🛫 any time to jump back to "now".
Single flight, no bulletin? Skip the paste — just type a flight number in the header pill (e.g. 27). If it matches a known ELY route the popup offers to create the leg with dep / arr already filled in.

📸 OPT / FMC screenshot reader OCR

Skip the typing — feed it the picture you already have.

Tap ⌖ OPT / FMC on the Takeoff performance group, pick a screenshot from Photos. The parser pulls out:

  • Tail from PROFILE EHB → normalised to 4X-EHB.
  • Flaps from the bottom output value.
  • N1 from TO2 93.2 (derate) or N1 92.5 (full thrust).
  • V1 / VR / V2 from the bottom-right block.
Powered by Tesseract.js. Cached on first use → works offline forever after that.

🧳 Per-leg state v7+

Leg 2's fuel doesn't bleed into leg 1. Ever.

Each leg owns its own data card and checklist. Switching legs swaps V-speeds, fuel, ATIS, SOB and checklist ticks with it. Crew + flight identity ride along automatically.

The 🛫 button next to the leg switcher arrows jumps to whichever leg's dep → arr window contains the current UTC.

🪪 Header — ELY flight number

Type digits (1, 27, 337) and the header auto-renders ELY###. Switching to a new number:

  • Matches a leg you already have → it switches to it.
  • Brand new → confirm popup shows route + typical block time and creates a new leg.

The ↗ next to the CTOT pill opens FlightAware for the current ELY ### so you can look up real ETD / slot.

📣 PA library EN + HE

One window, two languages, every QRH-style scenario.

Default set: Welcome, Go around, Emergency, Ditching, Smoke / fire, Malfunction, Delays. Hebrew on top (RTL), English below (LTR). Edit mode adds:

  • A sticky chip row of @-tokens — tap to insert instead of typing the @ key.
  • ◀ ▶ reorder buttons.
  • Inline name editing — just type in the title.

Variables that auto-fill:

🪪

Identity

@cpt@fo@PU@flight@tail
🗺

Route

@dep@arr@flighttime
⏱

Live

@time@utc@date@tod
@tod resolves to morning / noon / afternoon / evening / night based on the local clock — and it's Hebrew-aware (בוקר / צהריים / אחר הצהריים / ערב / לילה).

📡 QR cockpit share offline

Tap the ▣ icon in the header → a QR of the active flight appears. The other pilot taps the small camera icon, points it at your screen, and the data card lands as a new leg in their list.

What ships in the QR: tail, flight #, dep / arr, flight time, V-speeds, N1, flaps, fuel, ATIS, SOB, crew, route. Checklist ticks stay local. No internet needed once the QR libraries are cached.

🪂 AirDrop full-state sync

Same QR sheet → Sync everything (AirDrop) exports the whole app state as JSON via the iOS Share Sheet. Import on the other device replaces local state — useful when you want every leg, every speech, and the template moved across in one shot.

🎨 Themes & rotation

Light · Dark · ASU green — cycle with the theme icon. Orientation lock is off so landscape works on iPhone; if your installed PWA doesn't rotate, remove the home-screen icon and re-Add to Home Screen so it picks up the current manifest.

Built as a single-page PWA — vanilla JS modules, no framework, service-worker cache-first. Open the app and once it's loaded once, everything works offline.
