Flight Card 737NG

An iPad / iPhone cockpit companion for the Boeing 737NG. Tracks per-leg V-speeds, fuel, ATIS, SOB and crew; runs a customisable preflight checklist; reads OPT / FMC screenshots; carries the El Al PA library in EN + HE — and all of it works offline as a Progressive Web App.

Open the app → Share link

Data card

Per-flight data, grouped so the right field is one tap away during a busy turn:

  • Souls on board — total head count.
  • ATIS — letter chip + tap-through to a live D-ATIS / METAR popup with red/blue read state.
  • Takeoff performance — V1, VR, V2, N1, Flaps. The Flaps cell is collapsed by default; tap to choose from 1/5/10/15/25. Flap 5 is painted blue (standard), anything else paints red.
  • Fuel — Trip + Block. Type tonnes (12.3 or 5) and they auto-convert to kg on blur.
  • Flight — Dep, Arr, Flight time in HH:MM (auto-formatted as you type).
  • Crew — CPT, FO, Purser + CC2…CC5. Carried forward into every new leg from the roster.
  • Every group + the whole card have a ↻ reset that wipes its values in one tap. Flight + Crew are kept across resets.

OPT / FMC screenshot import OCR

Tap ⌖ OPT / FMC on the Takeoff performance group, pick a screenshot from Photos, and the parser pulls out: tail (PROFILE EHB → 4X-EHB), Flaps, N1 (handles TO2 93.2 derate labels and D-TO-2 alike), and V1 / VR / V2. Powered by Tesseract.js, cached on first use.

Checklist

Three sections by default — Cabin, Papers & performance, Checklist. Customisable via the ✎ edit mode. Items tick with a tap; the card auto-collapses when everything is done.

Roster import

Paste a duty bulletin (Slip details / Cockpit format) into the New Flight modal. The parser builds a sorted leg list with dep / arr / flight # / tail / scheduled times, seeds crew names from the slip, and switches the data card to the first added leg.

Per-leg state v7+

Each leg owns its own data card and checklist. Switching legs swaps V-speeds, fuel, ATIS, SOB and ticks with it. Crew + identity fields ride along automatically. The 🛫 button jumps to the leg whose dep/arr window contains the current UTC.

Header pill — ELY flight number

Type a flight number (digits only — 1, 27, 337) and it renders as ELY1. Switching to a new ELY ###:

  • If a leg with that number already exists → switches to it.
  • If it's brand new → confirmation modal shows route + typical block time from a local table of common ELY routes, and creates a new leg.

↗ next to the CTOT pill opens FlightAware for the current ELY ### to look up real ETD / slot.

PA speeches EN + HE

Bilingual library in one window — Hebrew on top (RTL), English below (LTR). Default set: Welcome, Go around, Emergency, Ditching, Smoke / fire, Malfunction, Delays. Edit mode adds a sticky chip row of @-tokens you tap to insert without typing the @ key, ◀ ▶ reorder buttons, and an inline name input.

Variables:

Identity

@cpt @fo @PU @flight @tail

Route

@dep @arr @flighttime

Live

@time @utc @date @tod

@tod resolves to morning / noon / afternoon / evening / night based on the local clock — Hebrew-aware (בוקר / צהריים / אחר הצהריים / ערב / לילה).

QR share + scan cockpit-to-cockpit

Tap the ▣ icon in the header. The other pilot's device scans the QR with the built-in camera and gets the data card as a new leg in their list. Payload is data-card only (tail, flight, V-speeds, N1, flaps, fuel, ATIS, SOB, crew, route) — checklist ticks stay local. Works offline once the QR libraries are cached.

AirDrop full-state sync

Same QR icon → "Sync everything (AirDrop)" → exports the whole app state as JSON via the iOS Share Sheet. Import on the other device replaces local state. For when you want every leg, every speech, the template — not just one flight.

Themes & rotation

Light, dark, ASU green — cycle with the theme icon. Phone-orientation lock is off so landscape works on iPhone. After re-saving to the home screen the manifest picks up orientation: any.

Built as a single-page PWA — vanilla JS modules, no framework, service-worker cache-first. Open the app and once it's loaded once, everything works offline.
